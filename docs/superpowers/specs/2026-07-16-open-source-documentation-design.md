# 开源文档设计

## 目标

为 `stm32f103-freertos-peripheral-lab` 建立一套能支持开源展示、复现工程和个人复习/面试准备的文档。

## 受众与边界

- `README.md` 面向 GitHub 访客、同学和招聘者，说明项目做了什么、如何构建、已验证范围和项目结构。
- `docs/learning-guide.md` 面向项目作者，解释代码架构、关键设计选择、调试经验和可直接复述的面试要点。
- 文档只陈述已在 STM32F103ZET6 正点原子精英板上完成的功能。CAN 仅描述为芯片内部回环验证，不声称完成真实 CAN 总线通信。
- 不伪造运行图片或视频；README 为 B 站演示视频预留一个可替换链接位置。

## 文档结构

### README

README 使用中文介绍，保留 STM32、FreeRTOS、PWM、DMA 等通用英文术语。内容依次为：项目概览、功能清单、硬件与软件环境、运行方式、B 站演示视频占位、目录结构、任务/模块概览、CAN 验证边界、后续方向、许可证与致谢。

### 学习手册

学习手册按程序实际运行顺序组织：从 `main` 和 RTOS 对象创建，到 `InitTask` 的启动门槛，再到各任务与外设模块。每个重要模块说明职责、核心机制、代码入口和面试表达。独立汇总真实调试问题：堆空间不足、看门狗复位、事件标志自动清除、DMP 初始化失败、Flash 擦除耗时和 WS2812 时序。

### 许可证

新增 MIT `LICENSE`，仅覆盖项目作者新增的代码和文档。STM32 HAL、CMSIS、FreeRTOS 与 MPU/DMP 等第三方目录继续适用其各自的许可证。

## 可维护性要求

- 视频发布后只修改 README 的单一视频链接行。
- 后续增加真实 CAN 或其他外设时，同时更新 README 的“已验证能力”和学习手册的对应章节。
- 文档中的文件名和任务名称必须与仓库现状一致。
